◆ prio

uint8_t SrNews::prio

request priority.

0: default.

SR_PRIO_BUF: request will be buffered if send fails, and will be re-tried latter. Buffering still does not mean 100% guarantee, as the buffer of the SrReporter has an explicit capacity. Old requests will be discarded if the capacity is exhausted.

SR_PRIO_XID: request uses a different template XID than SrAgent.XID(), and the first field in the CSV is the alternate XID.

Note
prio can be bit-wise XOR-ed, multiple priority can be set at the same time.
