Detailed Description

Generic MQTT network stack implementation.

Implementation is based on MQTT specification v3.1.1 (http://docs.oasis-open.org/mqtt/mqtt/v3.1.1/mqtt-v3.1.1.html). It supports both plain MQTT and MQTT + TLS.

Note
Currently only QoS level 0 and 1 are implemented, QoS 2 is not yet supported for both PUBLISH and SUBSCRIBE.

◆ yield()

int SrNetMqtt::yield ( int  ms)

Draft horse method ought to be called periodically.

This method tried to recv() from the server with timeout ms milliseconds, and invoke any registered SrMqttAppMsgHandler if there is data received from corresponding topic. It also clears other control packets (e.g. SUBACK, PUBACK) in the receive buffer. In addition, it checks for the keep-alive interval expiration, and will ping() the server by sending MQTT PINGREQ control packet.

Parameters
msrecv() timeout in milliseconds.
Returns
0 on success, -1 on failure.
Note
The method returns 0 if recv() timed out. -1 indicates a network error, e.g., broken connection.
Because of current implementation limit, timeout ms has only second resolution. For example, both 200 ms and 1234 ms will be rounded to 1 second.
